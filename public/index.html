O que é Node.js?

  • É um Event Driver com I/O não bloqueante
  • Ele roda em C++
  • Ele interpreta seu código Javascript (usando o V8 do Google),
  • E tem acesso ao SO, por meio destes módulos em C++
  • Mas você coda em Javascript
  • É multiplataforma, roda em Windows, Linux, Mac
E o o que isto muda para desenvolvedores Frontend?
Consideremos o ambiente do Desenvolvedor Frontend

Vc roda Javascript no Browser, logo,
você fica restrito às API's que o Browser lhe oferece


  • File System
  • Application Cache
  • Local Storage
  • Draggable (Acesso a metadados)
  • User Media (Camera)
Com Node, nós temos acesso ao SO, isto significa:
  • full access ao hardware
  • e programando com o bom e velho Javascript

Ta legal, posso fazer tudo, mas continuo sem idéias.

Gulp methods

gulp.src();

Gulp example

gulp
  .src('./assets/scripts/**/*.js')
  .pipe(sourcemaps.init())
  .pipe(concat('vendor.js'))
  .pipe(uglify())
  .pipe(sourcemaps.write('.'))
  .pipe(gulp.dest('./public/scripts'))
  .pipe(browserSync.stream());
